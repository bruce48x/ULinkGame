# 玩法与架构设计

这份文档是样例游戏设计的总入口。各功能领域的详细设计放在 `docs/features/` 子目录中。

## 设计目标

样例要验证 ULinkGame 在一个轻量多人对战项目中的完整接入方式：

- 单机模式可以完全离线运行。
- 联机模式由服务端权威推进对局。
- 客户端只发送输入，服务端广播世界状态。
- 玩法规则只实现一份，并被单机和联机复用。
- 网络、界面、存档和部署细节不能污染玩法规则内核。

当前玩法参考轻量小球吞噬对战。刻意不做分裂、吐质量、病毒、尖刺、组队和复杂技能树，避免样例被玩法复杂度拖偏。

## 暂不实现

- 分裂。
- 吐质量。
- 病毒或尖刺。
- 组队。
- 持久技能树改造。
- 服务端回滚或预测校正重写。
- 房间运行时自动迁移。
- 没有独立设计说明的 Redis 实时路由。
- 超越两周的历史排行榜归档。
